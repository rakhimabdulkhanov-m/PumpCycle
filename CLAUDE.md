# PumpCycle — demo app

## What this is
Single-page demo web app for septic pumping companies. Public sales demo
with fictional seed data ("Hawkins Septic Co"). No real users, no real sending.

## Stack (fixed — never add libraries without asking me)
- Vite + React, plain JavaScript (no TypeScript)
- Tailwind CSS
- react-leaflet + OpenStreetMap + Esri World Imagery (satellite) tiles, both free / no API key
- State: React state + localStorage (key "pumpcycle-demo-v4"), seeded from src/data/seed.json
- Deploy: Cloudflare Pages via wrangler. One Pages Function: POST /api/lead → Telegram bot (token/chat_id from env vars, never hardcoded)
- No other backend, no database, no auth, no router

## Hard constraints
- SMS/email sending is SIMULATED in UI only
- No tests, no refactors, no new npm packages, no TypeScript migration
- Mobile-responsive required; pixel-perfection not required
- Plain readable code, small components

## Product: topbar + 3 tabs
Seed: ~70 customers (residential cycleMonths 36 + a few commercial restaurants
cycleMonths 3). No `type` field — commercial is the predicate cycleMonths <= 3.
Each customer has `phone` and optional `email` (both may be empty); seed gives all
commercial accounts both, ~half the residential an email, the rest phone-only.
Dates auto-shift forward in loadState so the demo never goes stale.

1. Map: two tile layers via LayersControl — Esri World Imagery (satellite,
   default, no key) + OSM, both attributed.
   Customer pins — green / yellow (due ≤60 days) / red (overdue).
   Click pin → card: name, address, phone (tel:), email (mailto:), tank size,
   last pumped, cycle, next due (auto-calculated), notes. Commercial cards get a
   "Commercial · Grease trap" badge + cycle line. Overdue cards add a small
   "reach out" strip with Call / Email links (only the contacts that exist), since
   overdue customers have no auto-reminder. Buttons: [Mark pumped today] [Edit].
   Legend bottom-left (solid white so it reads on satellite; hidden on mobile
   while placing a pin).
   Drop-lid-pin flow: FAB "+ Drop lid pin" (hidden while placing).
   - Desktop: blue draggable pin centered + side panel together.
   - Mobile (two-step): step 1 full-screen map with draggable pin, top hint,
     "Next →"; step 2 sheet with service type
     (Residential 3-Year / Commercial Grease Trap 90-Day) + name + optional
     address + Save pin.
   Save creates a normal customer via the same addCustomer logic (Edit /
   Mark pumped / reminders all work on it); pin is green and persists.
2. Due list: rows sorted by next due; filters Overdue/30/60/90.
   Search box filters by name/address (filter-as-you-type, on top of the
   selected filter). + Add customer.
   Top counters: "Overdue: N — $X", "Due in 30d: N — $X", "Reminders scheduled: N".
   Revenue = count × avg job price (default $450, editable in a small settings popover).
3. Reminders: queue of every still-relevant reminder, generated by available
   contact — an Email reminder only if the customer has an email, an SMS reminder
   only if they have a phone (both → two separate rows on their own dates; neither
   → no reminder). Already-sent items move to the Sent filter; overdue customers
   drop out of the queue (still red on Map/Due list, but a "remind before due" item
   no longer applies) → clean queue of due-now + future (no Send-now backlog wall).
   - Email: residential 60d before due, commercial 15d before due; status
     Scheduled/Sent; "Send now" (simulated) flips to Sent with today's date.
   - SMS: 14d before due, click-to-text. Desktop "Copy text"; mobile "Copy text"
     + "Text from my phone" (sms: link with prefilled body, hidden if no phone).
     Manual "Mark as sent" (SMS is never auto-sent). Caption: "One tap sends
     from your phone." (no auto-send promise — pending 10DLC).
   Message text adapts: commercial uses FOG/grease-trap/90-day/manifest wording,
   residential plain language; day counts computed from real values, not hardcoded.
Topbar: "PumpCycle" logo text • badge "Live demo — sample data" •
button "Get this for your company" → modal: 2-line offer ($500 setup + $99/mo,
live in a week, we import your data) + form (Name, Email or phone) + hidden
honeypot field + line "or just reply to my email". POSTs to /api/lead.

## Audience
Buyers are 50–60yo non-technical owners. Big readable type, high contrast,
light theme, obvious buttons, zero clutter. Realistic data, never lorem ipsum.
